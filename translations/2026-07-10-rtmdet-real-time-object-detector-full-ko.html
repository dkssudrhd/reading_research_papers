초록 전문 번역

본 논문에서 우리는 YOLO 계열을 능가하면서도 인스턴스 분할(instance segmentation), 회전 객체 탐지(rotated object detection) 같은 여러 객체 인식 과제로 쉽게 확장할 수 있는 효율적인 실시간 객체 탐지기를 설계하고자 한다. 더 효율적인 모델 구조를 얻기 위해, 큰 커널 depth-wise convolution으로 구성한 기본 블록을 사용하고 backbone과 neck이 서로 잘 맞는 용량을 갖도록 한 구조를 탐색한다.

또한 정확도를 높이기 위해 동적 레이블 할당(dynamic label assignment)의 매칭 비용을 계산할 때 soft label을 도입한다. 더 나은 학습 기법까지 결합한 결과물인 RTMDet는 NVIDIA 3090 GPU에서 300 FPS 이상으로 동작하면서 COCO AP 52.8%를 달성해 당시 주류 산업용 탐지기를 능가한다.

RTMDet는 tiny, small, medium, large, extra-large 크기로 여러 적용 환경에서 우수한 파라미터-정확도 절충을 제공하며, 실시간 인스턴스 분할과 회전 객체 탐지에서도 새로운 최고 성능을 달성한다. 이 실험 결과가 다양한 객체 인식 과제를 위한 범용 실시간 탐지기 설계에 새로운 통찰을 제공하기를 바란다. 코드와 모델을 공식 저장소에 공개한다.

원문 본문 전체 번역

1. 서론 (Introduction)

최적의 효율은 객체 탐지에서 언제나 가장 중요한 목표이며, 특히 자율주행, 로보틱스, 드론의 현실 세계 인지에서는 더욱 그렇다. 이를 위해 YOLO 계열은 one-stage 객체 탐지기의 정확도와 효율을 계속 개선하도록 다양한 모델 구조와 학습 기법을 탐색해 왔다.

본 보고서는 YOLO 계열의 한계를 더 밀어붙이고, RTMDet라 부르는 새로운 실시간 객체 탐지 모델군(Real-Time Models for object Detection)을 제안한다. RTMDet는 기존 연구가 충분히 탐구하지 않았던 인스턴스 분할과 회전 객체 탐지도 수행할 수 있다. 주요 개선은 큰 커널 depth-wise convolution을 통한 더 나은 표현과, 동적 레이블 할당에서 soft label을 사용한 더 나은 최적화에서 나온다.

구체적으로 먼저 backbone과 neck의 기본 블록에 큰 커널 depth-wise convolution을 사용해 전역 문맥을 포착하는 능력을 높인다. 그러나 기본 블록에 depth-wise convolution을 곧바로 넣으면 모델 깊이가 늘어 추론이 느려진다. 그래서 블록 수를 줄여 깊이를 낮추고, 폭을 넓혀 모델 용량을 보완한다. 또한 neck에 더 많은 파라미터를 배치해 backbone과 비슷한 용량을 갖게 하면 더 좋은 속도-정확도 절충을 얻을 수 있음을 관찰했다. 이 구조 수정 덕분에 RTMDet는 model re-parameterization에 의존하지 않고도 빠르게 추론한다.

학습 전략도 다시 검토했다. 데이터 증강, 최적화, 학습 schedule의 더 나은 조합에 더해, 기존 동적 레이블 할당 전략에서 정답 box와 예측을 매칭할 때 hard label 대신 soft target을 쓰면 개선될 수 있음을 실험적으로 확인했다. 이 설계는 고품질 매칭을 위한 cost matrix의 구별 능력을 키우고 레이블 할당 노이즈를 줄여 정확도를 높인다.

RTMDet는 적은 변경만으로 인스턴스 분할과 회전 객체 탐지로 확장된다. kernel 및 mask feature 생성 head를 추가하면 약 10%의 추가 파라미터만으로 인스턴스 분할을 수행한다. 회전 객체 탐지는 box regression 차원을 4에서 5로 늘리고 rotated box decoder로 바꾸면 된다. 일반 객체 탐지 데이터셋 사전학습은 항공 영상의 회전 객체 탐지에도 유익했다.

광범위한 실험으로 RTMDet를 검증하고 tiny/small/medium/large/extra-large 크기로 확장했다. RTMDet-tiny는 4.8M 파라미터로 1,020 FPS에서 41.1% AP를, RTMDet-s는 819 FPS에서 44.6% AP를 보고했다. 인스턴스 분할에서는 COCO validation에서 180 FPS와 mask AP 44.6%, 회전 객체 탐지에서는 DOTA v1.0에서 AP 81.33%를 얻었다.

2. 관련 연구 (Related Work)

효율적인 객체 탐지 신경망 구조

객체 탐지는 장면 속 객체를 인식하고 위치를 찾는 과제다. 실시간 적용에서는 two-stage 탐지기보다 anchor 기반 또는 anchor-free one-stage 탐지기가 주로 연구된다. 효율을 높이기 위해 수작업 설계나 신경망 구조 탐색으로 효율적인 backbone, 모델 scaling, multi-scale feature 강화가 연구되었다. 최근에는 배포 뒤 추론 속도를 높이기 위한 model re-parameterization도 사용된다. 본 논문은 큰 커널 depth-wise convolution을 포함한 새 기본 블록으로 backbone과 neck의 용량을 맞추는 전체 구조를 제안한다.

객체 탐지를 위한 레이블 할당

탐지기를 개선하는 또 다른 축은 레이블 할당과 학습 loss다. 초기 방법은 정답 box와 예측 또는 anchor를 비교할 때 IoU를 매칭 기준으로 사용했고, 이후에는 객체 중심 같은 기준도 추가했다. 보조 탐지 head는 학습을 빠르고 안정적으로 만드는 데 사용됐다. DETR의 Hungarian assignment에 영향을 받은 동적 레이블 할당은 수렴 속도와 정확도를 크게 개선했다. 기존 전략이 loss와 같은 cost function을 쓰는 것과 달리, 본 연구는 매칭 cost에 soft label을 사용해 고품질·저품질 매칭의 차이를 키우고 학습을 안정화한다.

인스턴스 분할과 회전 객체 탐지

인스턴스 분할은 관심 객체마다 픽셀 단위 mask를 예측한다. mask classification, top-down, bottom-up 등 여러 패러다임이 존재하며 최근에는 box를 사용하거나 사용하지 않는 one-stage 방식도 등장했다. 동적 kernel을 mask feature map에 convolution하는 계열을 바탕으로, RTMDet에 kernel prediction head와 mask feature head를 붙인다.

회전 객체 탐지는 위치와 범주에 더해 방향을 예측한다. 기존 연구는 회전에 따른 feature misalignment를 줄이는 특징 추출기와 다양한 회전 box 표현을 연구했다. 본 논문은 여기에 직교하는 접근으로, 일반 탐지기에 angle branch를 추가하고 GIoU를 Rotated IoU Loss로 바꾸는 최소 수정만 수행한다. 이를 통해 일반 데이터셋에서 학습한 고정밀 탐지기의 구조와 지식이 고정밀 회전 탐지의 토대가 됨을 보인다.

3. 방법론 (Methodology)

3.1 거시 구조 (Macro Architecture)

RTMDet는 전형적인 one-stage 탐지기다. 전체 구조를 backbone, neck, head로 나눈다. YOLO 계열은 보통 네 stage와 여러 기본 블록으로 구성된 CSPDarkNet을 backbone으로 쓴다. neck은 backbone의 multi-scale feature pyramid를 받아 top-down·bottom-up 전파로 강화한다. 마지막 detection head는 각 scale feature map에서 box와 범주를 예측한다. 이 구조는 일반·회전 객체에 공통으로 적용되고, kernel 및 mask feature head를 붙여 인스턴스 분할로 확장할 수 있다.

이 거시 구조의 잠재력을 충분히 활용하기 위해 먼저 더 강력한 기본 블록을 연구하고, 구조의 계산 병목을 분석해 backbone과 neck의 깊이·폭·해상도를 균형 있게 조정한다.

3.2 모델 구조 (Model Architecture)

기본 블록. 큰 유효 수용영역은 영상 문맥을 넓게 포착하므로 탐지·분할 같은 dense prediction에 유리하다. Dilated convolution이나 non-local block은 계산 비용이 크지만, depth-wise convolution을 이용하면 합리적인 비용으로 큰 커널을 쓸 수 있다. 따라서 CSPDarkNet 기본 블록에 5×5 depth-wise convolution을 넣어 수용영역과 문맥 모델링 능력을 높인다.

최근 탐지기에서 사용하는 re-parameterized 3×3 convolution은 추론 정확도를 거의 공짜로 높이는 것처럼 보이지만, 학습 속도 저하와 메모리 증가를 일으키고 low-bit quantization 뒤 오차도 커질 수 있다. 큰 커널 depth-wise convolution은 학습 비용과 양자화 오차 측면에서 더 단순한 선택이다.

폭과 깊이의 균형. 큰 커널 뒤 point-wise convolution이 추가되면 layer 수가 늘어 병렬 계산이 어렵고 추론이 느려진다. 이를 해결하기 위해 각 backbone stage의 블록 수를 줄이고 폭을 적당히 넓혀, 용량과 정확도를 유지하면서 병렬성과 추론 속도를 높인다.

Backbone과 neck의 균형. 여러 크기의 객체를 탐지하려면 multi-scale feature pyramid가 필수다. backbone을 크게 하거나 feature fusion이 많은 무거운 neck을 쓰면 계산과 메모리가 증가한다. 본 연구는 backbone의 계산 일부를 neck으로 옮기고 neck 블록의 expansion ratio를 키워 두 부분이 비슷한 용량을 갖게 함으로써 더 나은 계산-정확도 절충을 얻는다.

공유 detection head. 여러 scale에 완전히 별도 head를 쓰면 정확도는 좋지만 파라미터가 많다. RTMDet는 scale 사이 convolution 가중치는 공유하고 Batch Normalization(BN) layer는 scale별로 분리한다. 서로 다른 feature scale의 통계 차이는 별도 BN이 처리하므로 파라미터를 줄이면서 정확도를 유지한다. 추론 시 학습 통계를 바로 쓰는 BN은 Group Normalization보다 효율적이다.

3.3 학습 전략 (Training Strategy)

레이블 할당과 loss. 각 scale의 dense prediction을 정답 box와 매칭해야 한다. 본 연구는 SimOTA를 바탕으로 동적 soft label assignment를 제안한다. 전체 cost는 분류, 회귀, center prior cost의 가중합이며 기본 가중치는 각각 1, 3, 1이다.

이진 레이블로 분류 cost를 계산하면 분류 점수는 높지만 box가 틀린 예측이 낮은 cost를 얻거나 그 반대가 될 수 있다. 이를 해결하기 위해 예측과 정답 box의 IoU를 soft label로 사용한다. soft classification cost는 regression 품질에 따라 cost를 다시 가중하고, 이진 레이블이 만드는 불안정한 매칭을 줄인다.

GIoU regression cost는 최고·최저 매칭 사이 차이가 작아 품질을 구별하기 어렵다. 따라서 loss에 쓰는 GIoU와 달리 assignment cost에는 -log(IoU)를 사용해 낮은 IoU 매칭의 비용을 확대한다. center cost도 고정 영역 대신 거리 기반 soft cost를 사용한다.

Cached Mosaic와 MixUp. 여러 샘플을 섞는 증강은 강력하지만 매 iteration 여러 이미지를 읽어야 해 느리고, 실제 데이터 분포와 다른 noisy sample을 만든다. 이전 이미지를 cache하면 I/O 요구를 줄여 한 이미지 처리에 가까운 비용으로 혼합할 수 있다. cache 길이와 pop 방식으로 동작을 조절한다. 큰 cache와 random pop은 원래 방식에 가깝고, 작은 cache와 FIFO는 같은 영상을 다른 증강으로 가까운 batch에서 다시 쓰는 repeated augmentation과 비슷하다.

2단계 학습. 앞 280 epoch에는 cached Mosaic와 MixUp을 쓰되 회전·shear는 box annotation과 입력의 정렬을 깨뜨릴 수 있어 제외한다. 대신 한 sample에 섞는 이미지를 8개로 늘린다. 마지막 20 epoch에는 Large Scale Jittering(LSJ)으로 바꿔 실제 데이터 분포에 더 가까운 영역에서 fine-tuning한다. 학습 안정성을 위해 AdamW를 사용한다.

3.4 다른 과제로의 확장

인스턴스 분할. RTMDet-Ins는 RTMDet에 CondInst와 비슷한 kernel prediction head와 mask feature head를 추가한다. 네 convolution layer가 multi-level feature에서 8-channel mask feature를 만들고, kernel head는 instance마다 169차원 vector를 예측한다. 이 vector는 88, 72, 9 길이의 세 부분으로 나뉘어 세 dynamic convolution kernel을 구성한다. mask feature에 좌표 feature를 연결해 instance mask를 생성한다. 동적 할당의 soft region prior에는 box 중심 대신 mask 질량 중심을 쓰며 dice loss로 mask를 감독한다.

회전 객체 탐지. RTMDet-R로 바꾸는 과정은 세 단계다. regression branch에 각도 예측용 1×1 convolution을 추가하고, box coder를 회전 box용으로 바꾸며, GIoU loss를 Rotated IoU loss로 교체한다. 대부분 파라미터를 공유하므로 COCO 같은 일반 탐지 데이터로 사전학습한 RTMDet 가중치가 좋은 초기값이 된다.

4. 실험 (Experiments)

4.1 구현 세부

객체 탐지와 인스턴스 분할은 train2017 약 118K장, val2017 5K장의 COCO를 사용한다. Ablation model은 300 epoch 학습하고 bbox AP와 mask AP로 평가한다. 모든 객체 탐지·인스턴스 분할 모델은 NVIDIA A100 GPU 8장으로 학습했다.

객체 탐지 test에서는 NMS 전 score threshold를 0.001로 두고 top 300 box를 남긴다. Ablation의 metric 계산을 빠르게 할 때는 threshold 0.05와 top 100을 사용하며, 이 설정은 약 0.3 AP를 낮출 수 있다.

회전 탐지는 2.8K 항공 영상과 188K instance를 가진 DOTA를 사용한다. Single-scale에서는 원본을 1,024×1,024 patch로, overlap 256 pixel로 자른다. Multi-scale에서는 원본을 0.5, 1.0, 1.5배로 바꾼 뒤 overlap 500 pixel로 자른다. 평가는 VOC2007 방식 mAP에 rotated IoU를 적용한다.

모든 latency는 NVIDIA RTX 3090, TensorRT 8.4.3, cuDNN 8.2.0, FP16, batch 1에서 측정했다.

4.2 Benchmark 결과

객체 탐지. YOLOv5, YOLOX, YOLOv6, YOLOv7, PPYOLOE와 비교했다. 공정성을 위해 모두 distillation·pruning 없이 300 epoch 학습했고 NMS 시간은 latency에서 제외했다. RTMDet-tiny는 4.8M 파라미터로 AP 41.1을 달성해 비슷한 크기 모델보다 5 AP 이상 높았다. RTMDet-s는 YOLOv6-s의 절반가량 파라미터·계산량으로 더 높은 정확도를 보였다. RTMDet-m과 l은 각각 AP 49.4와 51.5, RTMDet-x는 AP 52.8과 300 FPS 이상을 기록했다.

인스턴스 분할. 먼저 ResNet50-FPN과 표준 multi-scale schedule에서 CondInst보다 mask AP가 1.5 높았다. tiny/s/m/l/x 모델을 같은 300-epoch 설정으로 학습한 결과 RTMDet-Ins-x는 mask AP 44.6으로 YOLOv5-seg-x보다 3.2 높으면서 실시간 속도를 유지했다.

회전 객체 탐지. DOTA v1.0 single-scale에서 RTMDet-R-m과 l은 각각 mAP 78.24와 78.85였다. Multi-scale에서는 80.26과 80.54로 올랐고, COCO 사전학습 RTMDet-R-l은 mAP 81.33을 기록했다. 특수 operator가 없어 여러 하드웨어에 배포하기 쉽다는 점도 강조한다.

4.3 모델 구조 Ablation

큰 커널. 3×3, 5×5, 7×7을 비교해 3×3 convolution과 5×5 depth-wise convolution의 조합이 최적 절충이었다. 7×7은 AP가 조금 높지만 latency 증가가 컸다.

Feature scale 균형. 두 번째·세 번째 stage 블록을 9개에서 6개로 줄이면 latency가 20% 줄고 AP가 0.5 낮아졌다. 각 stage 끝에 Channel Attention을 넣어 감소를 0.1 AP로 줄이면서 latency를 7% 개선했다.

Backbone과 neck. 계산을 backbone보다 neck에 배치해 둘의 용량을 비슷하게 만들면 작은·큰 모델 모두 비슷한 정확도에서 더 빠른 속도를 얻었다.

Detection head. 완전 공유 head는 scale별 통계 차이 때문에 성능이 낮았다. 완전히 별도 head는 정확하지만 파라미터가 늘었다. convolution 가중치를 공유하고 BN만 분리한 head가 가장 좋은 절충을 보였다.

4.4 학습 전략 Ablation

ResNet-50 baseline의 AP 39.9에서 분류 cost에 IoU soft label을 넣어 40.3, soft center prior로 40.8, log-IoU cost로 41.3까지 높였다. 동일 구조·loss에서 다른 할당 전략보다 최대 0.5 AP 높았고, 긴 schedule과 강한 증강에서는 RTMDet-s에서 SimOTA보다 1.3 AP 높았다.

첫 280 epoch와 마지막 20 epoch의 증강 조합을 비교했다. Cache는 충분한 sample에서 원래 Mosaic/MixUp과 같은 효과를 내면서 Mosaic을 약 3.6배, MixUp을 약 1.5배 빠르게 했다. 마지막 단계에 LSJ를 사용하면 두 단계 모두 Mosaic/MixUp을 쓰는 것보다 RTMDet-s에서 2 AP, RTMDet-l에서 1.5 AP 높았다. 작은 cache와 FIFO는 tiny/small에서 약 0.5 AP 개선될 수 있었다.

강한 증강에서 SGD는 불안정했으므로 AdamW, weight decay 0.05, Cosine Annealing을 baseline으로 선택했다. 학습 전반부에는 고정 학습률, 후반부에는 cosine annealing을 쓰는 Flat-Cosine이 0.3 AP를 높였다. normalization layer와 bias에는 weight decay를 적용하지 않아 0.9 AP, RSB 방식 ImageNet 사전학습 backbone으로 0.3 AP를 추가로 얻었다.

4.5 단계별 결과

YOLOX-s baseline AP 40.2에서 AdamW와 Flat-Cosine으로 40.6, 새 구조로 41.8이 되었다. SepBNHead는 정확도를 유지하며 파라미터를 10.07M에서 8.89M으로 줄였다. label assignment와 loss로 42.9, 개선한 증강으로 44.2, RSB 사전학습으로 44.5가 되었다. 모든 변경의 상승폭은 baseline 대비 4.3 AP다.

5. 결론 (Conclusion)

본 논문은 모델 구조, 레이블 할당, 데이터 증강, 최적화를 포함해 실시간 객체 탐지기의 각 핵심 요소를 포괄적으로 실증 분석했다. 또한 고정밀 실시간 탐지기를 인스턴스 분할과 회전 객체 탐지로 최소 수정해 확장하는 방법을 탐구했다. 그 결과 RTMDet와 여러 객체 인식 과제용 파생 모델군을 만들었다. RTMDet는 다양한 모델 크기로 산업용 적용에서 정확도와 속도의 우수한 절충을 보인다. 이 결과가 실시간 객체 인식 연구와 산업 발전의 길을 넓히기를 바란다.

부록 A. Benchmark 결과

A.1 추가 비교

PPYOLOE-R 비교. 같은 환경에서 RTMDet-R과 PPYOLOE-R을 비교했다. RTMDet-R-m과 l은 PPYOLOE-R-l과 x보다 정확하면서 각각 18.5%, 20.8% 빠르다고 보고했다. RTMDet-R 코드와 모델은 MMRotate에 공개했다.

DOTA-v1.5. DOTA-v1.0과 같은 영상에 10 pixel 미만의 극소 객체 215K개를 추가한 데이터셋이다. NVIDIA A100 GPU 4장으로 학습하고 COCO 사전학습을 기본 사용했다. RTMDet-R-l은 ReDet보다 mAP 1.32 높았다.

HRSC2016. Google Earth에서 수집한 약 1K 영상과 2.9K 선박의 데이터셋이다. 영상 aspect ratio를 바꾸지 않고 108 epoch 학습했다. RTMDet-R-tiny는 mAP07 90.6과 mAP12 97.1을 보고했다.

번역자 메모: 부록의 대형 class별 수치표는 원 데이터의 의미를 바꾸지 않도록 전체 전사를 생략하고 주요 비교와 RTMDet 행을 설명했다. 정확한 class별 AP는 원문 Tables A1–A3을 확인해야 한다.

수식과 기술 용어

Equation 1 · 전체 매칭 비용

C = λ₁Ccls + λ₂Creg + λ₃Ccenter

분류, box 회귀, 중심 prior 비용의 가중합이다. 기본값은 λ₁=1, λ₂=3, λ₃=1이다.

Equation 2 · Soft classification cost

Ccls = CE(P, Ysoft) × (Ysoft − P)²

Ysoft는 예측 box와 정답 box의 IoU다. 분류 확률과 위치 품질이 함께 좋은 예측을 선호한다.

Equation 3 · Regression cost

Creg = −log(IoU)

낮은 IoU에서 비용 차이를 크게 만들어 나쁜 매칭을 더 명확히 배제한다.

Equation 4 · Soft center region cost

Ccenter = α / (|xpred − xgt| − β)

원문의 표기를 보존했다. α=10, β=3을 기본값으로 두며 고정된 3×3 중심 영역보다 부드러운 위치 prior를 만든다.

Depth-wise convolution채널별 공간 convolution. 큰 kernel의 계산량을 낮춘다.
