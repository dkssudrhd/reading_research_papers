Abstract — 초록

초저전력 장치용 TinyML은 센서와 모델을 함께 배치하지만, 작은 모델은 라벨 잡음에 특히 민감하고 맞춤형 이진 과제에는 표준 벤치마크가 부족하다. 이전 person detection 표준인 Visual Wake Words(VWW)는 약 123K 이미지와 추정 라벨 오류율 7.8%를 가져, 제품 수준 모델 학습에 한계가 있다.

수동 주석은 TinyML 사용 사례가 요구하는 규모와 다양성에서 비용이 지나치게 크다. 저자들은 Wake Vision 파이프라인으로 이 간극을 다룬다. 이 파이프라인은 image-level과 bounding-box 라벨의 결합, confidence·면적·묘사물 인지 필터, 평가 split의 라벨 오류 정정, 세분 benchmark subset의 자동 생성을 결합한다.

person detection에 적용한 결과 Wake Vision은 약 600만 이미지, 수동으로 다시 라벨링한 validation/test, 2.2% 라벨 오류율을 제공한다. MobileNetV2, MCUNet, MicroNets, ColabNAS에서 VWW 대비 최대 6.6%의 test accuracy 향상을 보였고, 성별·연령·거리·조도·묘사물의 16개 세분 subset 중 13개에서 같거나 더 높은 성능을 보였다. 모든 artifact는 TFDS와 Hugging Face를 통해 CC BY 4.0으로 공개된다.

본문 전문 번역

1 Introduction — 서론

초저전력 장치에 머신러닝을 배치할 때 에너지 소비와 비용을 줄이기 위해 TinyML 분야가 발전했다. TinyML은 항상 켜진 microcontroller(MCU)와 TinyML accelerator에 모델을 배치하여, 데이터 전송이나 큰 프로세서를 계속 구동하는 비용을 줄인다. 그러나 이 환경의 모델은 수백 KiB 수준으로 제한될 수 있어 모바일폰보다 약 네 자릿수 적은 메모리를 쓴다.

TinyML에는 고품질 연구를 위한 대규모 데이터셋이 필요하지만, 하드웨어 제약은 모델을 작고 효율적으로 만들므로 전통적이고 복잡한 과제에 맞지 않는다. ImageNet 같은 범용 데이터셋은 이 목적에 잘 맞지 않을 수 있다. 기존 TinyML 데이터셋도 유용했지만 규모가 제한되어 제품급 모델을 단독으로 학습하기 어렵다.

저자들은 TinyML용 person detection의 대규모 데이터셋 Wake Vision을 제시한다. person detection은 사람이 있는지를 판단하는 이진 이미지 분류 과제다. Wake Vision은 기존 VWW보다 약 100배 큰 600만 장 이상 이미지로 구성되며, Large와 Quality 두 학습 변형을 제공한다. Large는 image-level label을 이용해 규모를 우선하고, Quality는 bounding box 기반으로 더 신뢰도 높은 라벨과 필터링을 우선한다.

논문은 저조도·먼 거리처럼 인터넷 이미지에서 드문 실제 배치 조건을 따로 평가하기 위해 세분 benchmark도 제안한다. 또한 입력 크기, 모델 크기, INT8 quantization, grayscale 변환이 이들 조건에 미치는 영향을 사례 연구로 분석한다. 기여는 대규모 허용 라이선스 데이터셋, 다섯(논문 버전에 따라 세부 조합으로 여섯) 세분 benchmark, 그리고 TinyML 설계의 세분 분석이다.

2 Related Work — 관련 연구

person detection의 기존 사실상 표준은 VWW다. 일반 데이터셋에서도 person label로 유사 모델을 만들 수 있으나, TinyML에 맞춰 공개·설계된 데이터는 드물다. Wake Vision은 기존 공개 데이터보다 훨씬 큰 규모와 fine-grained benchmark를 제공하며 TensorFlow Datasets 및 Hugging Face로 배포된다.

데이터 품질은 작은 모델에서 특히 중요하다. 지식 증류(knowledge distillation)는 큰 noisy dataset의 정보를 활용할 수 있지만, 라벨·샘플 필터링도 필요하다. 저자들은 일반 정확도만으로 실제 배치 품질을 판단하지 않고, 거리·조도·묘사물·인구통계 proxy 조건별 점검을 제안한다.

3 Wake Vision Dataset — Wake Vision 데이터셋

Wake Vision은 TinyML person detection용 대규모 데이터셋이다. Open Images V7에서 유래하며, 고용량 Large와 고품질 Quality의 두 training set을 제공한다. 원문 표의 Quality는 총 1,423,690장(학습 675,411장, validation 9,106장, test 27,328장)으로, Large는 6,477,906장(학습 3,238,953장)으로 제시된다. 버전별 갱신에 따라 정확한 split 수는 원문과 데이터 카드에서 재확인해야 한다.

3.1 Label Generation — 라벨 생성

수백만 장을 직접 주석하는 비용을 피하기 위해 Open Images의 기존 라벨을 사용한다. 기본 라벨은 person/non-person 이진 라벨이다. image-level label은 객체의 존재만, bounding box label은 위치 좌표도 제공한다. bounding box label은 더 정확하고 필터링 정보가 많지만 수가 적다. Open Images training에는 사람 image-level label이 300만 장 이상 있지만 사람 bounding box가 있는 이미지는 약 70만 장이다.

따라서 더 많은 데이터와 더 높은 라벨 품질 사이의 trade-off가 생긴다. Large는 image-level label을 사용하고, Quality는 bounding box label을 사용한다. 저자들의 실험에서 Quality가 Large보다 정확한 모델을 만들었으며, 이 결과는 해당 설정에서 수량보다 품질이 중요함을 시사한다.

3.1.1 Large Scale Training Set from Image-Level Labels — 이미지 수준 라벨의 대규모 학습 집합

image-level label에서 추가로 유용한 값은 confidence다. 기계 생성 confidence는 0–1이고 사람 검증 라벨은 0 또는 1이다. 저자들은 낮은 confidence의 person label만 가진 이미지를 제외하여 Large 집합을 만든다.

3.1.2 High Quality Training Set From Bounding Box Labels — bounding box 라벨의 고품질 학습 집합

bounding box label은 사람이 검증하고 위치를 표시하므로 false positive 가능성이 낮다. box 면적으로 사람이 이미지에서 차지하는 비율을 근사해, 사람이 아주 멀리 있어 매우 작은 경우를 Quality 학습 집합에서 제외한다. 또한 depiction flag를 이용하여 그림 속 사람을 기본적으로 person으로 보지 않도록 한다. 이러한 옵션은 데이터 생성 script로 조정할 수 있다.

3.1.3 Training Set Comparison — 학습 집합 비교

동일한 MobileNetV2-0.25를 같은 step 수로 학습했을 때 Quality는 Large보다 높은 test accuracy를 보였다. 원문 표에서 Large는 86.00%, Quality는 88.65%였고, MobileNetV2-1.0 teacher를 이용한 distillation 뒤에는 각각 88.96%, 89.32%였다. 즉 soft label만으로 차이가 사라지지 않아, bounding-box 기반 필터가 데이터 자체의 품질을 높였음을 시사한다.

3.2 Error Correction — 오류 정정

Wake Vision 라벨은 Open Images 파이프라인의 오류를 물려받을 수 있다. 저자들은 confident learning으로 validation/test의 오류 후보를 찾고 사람이 확인해 정정한다. 후보 중 실제 오류는 12–16%였지만, 이 절차로 평가 데이터의 신뢰성을 높였다. 표본 평가에서 VWW test의 최상위 라벨 오류율은 7.8%, Wake Vision은 6.8%였으며 depiction 관련 오류를 제외하면 Wake Vision 오류는 더 낮았다.

3.3 Usability — 사용성

데이터셋은 TFDS와 Hugging Face에서 제공된다. 이미지와 라벨을 재호스팅해 링크 소멸로 데이터가 바뀌는 문제를 줄인다. validation/test label CSV는 이미지 이름을 key로 하며, depiction flag와 distance·lighting 등 benchmark inclusion flag도 제공한다. 필요한 부분만 내려받고 갱신하기 쉬운 구조다.

3.4 Comparison to Visual Wake Words — VWW와 비교

동일 recipe의 MobileNetV2를 VWW와 Wake Vision에서 각각 학습해 교차 평가했다. 원문 표에서 VWW test는 VWW 학습 88.00%, Wake Vision 학습 88.50%였고, Wake Vision test는 각각 86.27%, 88.68%였다. Wake Vision이 단순 도메인 이동이 아니라 VWW test에서도 직접적인 향상을 준다는 해석이다.

4 Fine-Grain Benchmark Suite — 세분 benchmark 모음

인터넷 공개 이미지로 만든 test set은 실제 사용 조건을 충분히 대표하지 않을 수 있다. 사람 탐지 시스템은 저조도처럼 더 어려운 환경에서도 동작해야 한다. 저자들은 설계 단계에서 견고성을 분석하기 위해 perceived gender, perceived age, subject distance, lighting, depictions를 분리한 benchmark를 만들었다.

각 benchmark는 배치 목적에 따라 모델 선택을 돕는다. 예를 들어 가까우며 밝은 대상과 멀고 어두운 대상은 전혀 다른 설계 결정을 요구할 수 있다. Wake Vision 모델은 VWW 모델보다 여러 어려운 subset에서 더 견고했으며, 예를 들어 depiction 사람에서 F1 0.83 대 0.77, non-person depiction에서 0.89 대 0.85를 보였다. 연령 subset에서도 Wake Vision이 더 높은 F1을 보고했다.

5 Model Design Case Study — 모델 설계 사례 연구

저자들은 세분 benchmark를 이용해 입력 크기, 모델 크기, INT8 양자화, grayscale의 효과를 조사한다. 전체 test accuracy만 보면 input scaling과 model scaling의 영향이 비슷해 보일 수 있다. 그러나 먼 사람 subset만 보면 입력 이미지 크기를 키우는 효과가 훨씬 커서, 먼 대상에서는 입력 해상도가 병목임을 알 수 있다.

5.1 Quantization — 양자화

INT8 quantization은 모든 benchmark에서 성능 저하가 거의 없었고(±0.004 F1), 지연·메모리·모델 크기 측면의 이점을 제공했다. 저자들은 person detection에서는 양자화가 이득이라고 결론내린다. 다만 이는 해당 과제·데이터에서의 관찰이며, 다른 대상·카메라에는 별도 검증이 필요하다.

5.2 Grayscale — 그레이스케일

RGB를 grayscale로 바꾸면 메모리를 크게 줄일 수 있다. 그러나 원문은 grayscale 모델이 특히 밝은 이미지에서 더 민감하다고 보고한다. 평균 지표만으로는 이런 운영 조건의 악화를 놓칠 수 있다.

6 Ethical Considerations — 윤리적 고려

Wake Vision은 TinyML 연구와 에너지 절감형 person detection을 돕지만, 사람 탐지는 무기 표적화나 대규모 감시처럼 악용될 수 있다. 이미지는 Flickr의 CC BY 2.0으로 표시된 항목을 Open Images를 통해 사용했으나, 저자들은 각 이미지의 라이선스 상태를 보증하지 않는다고 명시한다. 인구통계 benchmark도 공정성·견고성 확인을 위한 것이며, 성별·연령 분류 시스템으로 오용될 수 있다.

7 Conclusions — 결론

저자들은 TinyML이 고품질 대규모 데이터 부족으로 막혀 있다고 보고 Wake Vision을 제안했다. Wake Vision은 VWW보다 약 100배 크고, 더 사용하기 쉽고, 라벨 오류가 비슷하거나 낮으며 실제 조건을 더 잘 반영한다. 세분 benchmark는 공정성·어려운 조건의 견고성을 평가하게 하고, 입력·모델 scaling과 양자화 같은 설계를 평균 점수 너머에서 분석하게 한다.

수식·기술 용어

이 논문은 새 손실함수의 수식보다 데이터 생성·검증 프로토콜 중심이다. 중요한 표기와 용어는 다음과 같다.

  • F1 = 2PR / (P + R): precision(P)과 recall(R)의 조화평균. 작은 새에는 전체 F1 외 거리·조도 bucket별 F1이 필요하다.
  • INT8: 8-bit 정수 양자화. 낮은 메모리·지연을 얻지만 배치 대상 데이터에서 재측정해야 한다.
  • teacher → student: 지식 증류. 큰 teacher의 확률 출력(soft label)을 작은 student가 학습한다.
  • image-level label: 객체 존재만 표시한 라벨, bounding-box label: 위치까지 표시한 라벨.
  • confident learning: 예측과 관측 라벨의 불일치로 오류 후보를 찾는 데이터 정제 방법.

번역자 메모 3: 참고문헌은 원문 PDF의 서지 목록을 따르며 이 페이지에서는 반복 번역하지 않았다.

실무 독해 메모

  • bird detection: 객체 면적·거리·배경별로 validation을 고정하고 평균 AP 하나로 모델을 고르지 않는다.
  • small/tiny object: 먼 새의 누락은 backbone 확장보다 입력 해상도·tile 정책이 먼저일 수 있다.
  • PTZ camera tracking: detect recall과 track 유지율, PTZ 재획득 시간을 각각 거리 bucket별로 측정한다.
  • real-time RTSP/video: decoder 지연과 queue drop을 모델 latency와 분리해 timestamp 기반으로 기록한다.
  • YOLO training/data: pseudo-label 대량 풀과 사람 검수 Quality 풀을 분리해 label-noise ablation을 한다.
  • ONNX/TensorRT/edge: FP16/INT8은 전체 mAP만 아니라 야간·역광·먼 새 recall을 release gate로 둔다.
  • Hanwha Vision: RTSP profile·해상도·bitrate와 PTZ preset별 대상 pixel 크기를 메타데이터로 남겨 평가 subset을 자동화한다.
