한국어 초록

작은 새처럼 빠르게 움직이는 여러 객체를 UAV 영상에서 추적하는 일은 어렵습니다. 대상은 작아서 appearance feature가 거의 없고, 카메라와 새가 동시에 움직여 motion이 뒤섞이며, 무리 행동 때문에 occlusion과 ID ambiguity가 자주 생깁니다. 이 논문은 tracking-by-detection 구조를 유지하면서 검출 단계에서는 SliceTrain으로 작은 객체 학습을 강화하고, 추적 단계에서는 OC-SORT에 EMA 기반 motion direction 유지와 bounding box expansion, distance penalty를 더합니다. SMOT4SB public test에서 SO-HOTA 55.205를 달성하며 작은 새 추적을 위한 실용적 기준선을 제시합니다.

섹션별 상세 읽기

1. Introduction

저자들은 UAV 시점의 작은 새 추적을 일반 MOT보다 어려운 문제로 정의합니다. 핵심 난점은 외형 정보 부족, 카메라와 대상이 동시에 움직이는 motion entanglement, 새 무리에서 생기는 occlusion과 ID switch입니다.

2. Related Work

일반 MOT는 사람이나 차량처럼 크고 외형 단서가 풍부한 대상을 많이 가정합니다. 작은 새는 texture와 색상 단서가 약하므로 Re-ID보다 motion association을 어떻게 안정화할지가 중요합니다.

3.1 Detector

검출기는 YOLOv8 기반입니다. SliceTrain은 고해상도 이미지를 겹치는 tile로 빠짐없이 자르고, tile마다 독립적인 증강을 적용해 작은 새가 학습 중 더 자주, 더 크게 보이도록 만듭니다.

3.1.2 Full Size Inference

학습은 tile로 하지만 추론은 원본 full-size frame에서 수행합니다. 타일 추론과 병합 비용을 피하면서 작은 객체에 민감해진 detector의 장점을 가져가려는 비대칭 전략입니다.

3.2 Tracker

추적기는 appearance feature에 의존하지 않고, OC-SORT/ByteTrack 계열의 관측 기반 association을 강화합니다. EMA로 방향을 부드럽게 유지하고, 작은 box에서 IoU가 쉽게 0이 되는 문제를 expansion과 center distance로 보완합니다.

4. Experiments

SMOT4SB 데이터셋에서 YOLOv8-L/M/S를 비교합니다. YOLOv8-L은 가장 높은 SO-HOTA를 보이고, YOLOv8-S는 FPS가 높아 실시간 운영 후보로 읽을 수 있습니다.

4.4 Ablations

기본 OC-SORT에 EMA, bbox expansion, distance penalty를 순서대로 더할 때 SO-HOTA가 단계적으로 상승합니다. 이는 작은 객체 추적 실패 원인을 하나씩 줄인 결과로 해석됩니다.

5. Conclusion

논문은 작은 새 추적에서 detector 학습과 tracker association을 동시에 다뤄야 한다고 결론짓습니다. 다만 극단적인 motion이나 긴 occlusion에서는 motion heuristic의 한계가 남습니다.

중요 용어

SMOT (Small Multi-Object Tracking)

작게 보이는 여러 객체를 프레임 사이에서 지속적으로 연결하는 문제입니다. 이 논문에서는 UAV 영상 속 작은 새가 대상입니다.

SliceTrain

고해상도 이미지를 겹치는 tile로 자르고 tile마다 강한 증강을 적용하는 학습 데이터 구성 방법입니다.

OC-SORT

관측 중심의 online multi-object tracker입니다. 이 논문은 작은 새 조건에 맞게 motion 방향과 유사도 metric을 보강합니다.

EMA (Exponential Moving Average)

최근 motion 방향을 부드럽게 누적하는 방식입니다. sudden turn이나 검출 jitter에 track ID가 쉽게 흔들리지 않도록 돕습니다.

SO-HOTA

작은 객체 tracking을 위한 HOTA 계열 지표입니다. 작은 bbox에서는 IoU가 너무 민감해 Dot Distance 기반 평가를 사용합니다.

Distance penalty

IoU만 보지 않고 중심점 이동 거리를 함께 고려하는 유사도 보정입니다.

실무 메모

  • bird detection: 작은 새는 appearance가 약하므로 detector recall을 먼저 끌어올리고 confidence threshold를 tracking 정책과 함께 튜닝합니다.
  • small/tiny object detection: SliceTrain은 학습 배치 안의 작은 객체 정보 밀도를 높입니다. SAHI 추론보다 운영 비용이 낮을 수 있습니다.
  • PTZ camera tracking: EMA 방향 유지와 center distance penalty는 PTZ pan/tilt 명령 smoothing에 직접 연결됩니다. track loss에는 짧은 hold window를 둡니다.
  • real-time RTSP/video analysis: decode, resize, detector, tracker, PTZ command까지 end-to-end latency를 함께 재야 합니다.
  • YOLO training/data design: 원본 고해상도 라벨을 유지하면서 1280급 tile을 만들고, tile 좌표와 원본 좌표 변환을 재현 가능한 스크립트로 고정합니다.
  • ONNX/TensorRT/edge server deployment: YOLOv8-S/M/L별 TensorRT FP16/INT8 후보를 만들고, tracker는 CPU에서도 충분한지 분리 측정합니다.
  • Hanwha Vision camera, RTSP, PTZ integration: Hanwha Vision 광각 RTSP에서 track ID를 유지하고, PTZ는 가장 위험한 track 하나만 따라가게 합니다. ONVIF 명령 반응시간과 영상 지연을 함께 기록해야 합니다.
