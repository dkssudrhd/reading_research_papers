풍력단지 조류 탐지를 위한 Temporal Boosted YOLO

4K 흑백 영상에서 작은 비행 조류를 검출하기 위해 YOLOv4, 타일링(tiling), 시간적 프레임 스태킹(temporal stacking), 앙상블을 단계적으로 결합한 연구다.

summary · 한눈에 보는 요약

문제

하늘처럼 넓은 4K 화면에서 새는 몇십~몇백 픽셀로만 보이고, 구름·조도·카메라 노출이 계속 변한다.

해결

전체 프레임을 네 타일로 나누고, 기준 프레임의 앞·뒤 프레임을 함께 입력해 작은 새의 픽셀 정보와 움직임 단서를 보존한다.

결과

Klim 테스트에서 Model 1 82.9%, Model 2 88.7%, Model 3 90.0% mAP를 얻었다. 6개 Model 3의 affirmative 앙상블은 94% mAP였다.

해석 주의: 논문의 mAP는 VOC 계열과 AP50 중심이며, 연구용 4K 흑백 영상과 특정 데이터 분할에서 얻은 값이다. 현장 RGB PTZ 카메라의 end-to-end 성능으로 그대로 간주하면 안 된다.

why · 왜 오늘 이 논문인가

월요일의 조류 탐지·풍력단지 주제와 직접 맞고, 현재 업무의 세 가지 병목을 한 논문에서 다룬다. 첫째, 입력을 축소할 때 작은 새의 정보가 사라지는 문제다. 둘째, 단일 프레임 모델이 모션 블러와 초점 변화에 취약한 문제다. 셋째, 구름이 움직이는 하늘에서 background subtraction이 새가 아닌 영역을 대량으로 오검출하는 문제다.

또한 Skagen과 Klim이라는 실제 야외 영상 데이터셋을 소개하고, 공식 데이터 저장소와 데이터 카탈로그를 연결한다. 원문은 CC BY 4.0이므로 출처·라이선스·변경 사실을 표시한 번역본을 저장할 수 있다.

core-idea · 핵심 아이디어

공간 정보: tiling

4K 전체를 네 조각으로 나누면 네트워크 입력 크기로 리사이즈해도 새가 차지하는 상대적 픽셀 수가 커진다. 단, 타일 경계에 걸친 새를 놓칠 수 있다.

시간 정보: stacking

t−1, t, t+1 영상을 채널처럼 쌓는다. 별도 tracker 없이도 움직임·블러·크기 변화 단서를 학습한다.

technical-details · 기술 세부

Model 1

4K 프레임 전체를 YOLOv4 입력 크기로 줄인다. 구조는 CSPDarknet-53 backbone, SPP, PANet neck, YOLOv3 head다.

Model 2

4K 프레임을 4개로 분할하고 각 타일을 독립적으로 추론한 뒤 좌표를 원본으로 합친다.

Model 3

세 시점의 4K 프레임에서 각각 4개 타일을 만들어 총 4개의 시간 스택 입력을 구성한다. 논문은 tracking 정보를 직접 사용하지 않는다.

학습 설정

COCO 사전학습 YOLOv4 가중치에서 전이학습하고, batch 64, subdivision 32, 1024 입력, momentum 0.949, learning rate 0.001, 9개 anchor를 탐색했다.

배경 차분 baseline은 background model → morphological gradient → Gaussian blur(9×9) → blob detection 순서다. Skagen처럼 배경이 비교적 고정된 곳에서는 precision 84%, recall 60%를 보였지만, 구름과 조도가 변하는 Klim에서는 오검출이 크게 늘었다.

application · 조류충돌방지 시스템 적용

bird / tiny detection

자사 영상에서 전체 프레임·4-tile·overlap tile을 비교하고 16, 32, 64px 구간별 recall을 기록한다. 타일 경계 새는 overlap과 NMS로 보완한다.

PTZ tracking

Model 3의 검출을 OC-SORT 또는 ByteTrack에 연결한다. track age와 confidence가 안정된 뒤 PTZ를 움직여 검출-제어 진동을 줄인다.

RTSP 운영

Hanwha Vision RTSP decode → timestamp 보존 → tile/stack 전처리 → YOLO/TensorRT → tracker → PTZ command의 p50·p95 지연을 각각 기록한다.

YOLO 데이터·학습

논문처럼 site를 넘나드는 일반화를 검증하려면 한 장소를 train에 섞지 않고 site-level holdout을 둔다. RGB·흑백, 구름·역광·비·안개, blade·insect·drone hard negative를 별도 태그로 관리한다. 단일 mAP보다 false alarm/hour, missed-bird rate, track fragmentation을 함께 본다.

ONNX/TensorRT·엣지 서버

타일 수가 늘면 정확도뿐 아니라 GPU 메모리, decode backlog, PTZ 제어 지연이 늘어난다. 먼저 한 타일 FP16 baseline을 만들고, 2·4타일 및 temporal stack을 단계적으로 활성화한다. Jetson TX2라는 논문의 엣지 사례는 현장형 설계의 출발점이지만 현재 장비에서는 재벤치마크해야 한다.

experiment · 바로 실행할 3가지

  1. 해상도·타일 ablation: 같은 1시간 RTSP 구간에서 full-frame YOLO, 2×2 tile, 2×2 overlap tile을 비교하고 작은 새 recall·GPU latency·false alarm/hour를 표로 만든다.
  2. 시간 스택 ablation: 단일 프레임, RGB 3채널 대용 시간 스택, 5프레임 입력을 비교한다. PTZ 이동 중 blur 구간과 정지 구간을 분리 평가한다.
  3. 검출-PTZ 폐루프: 검출 confidence·track age·화면 중심 오차에 hysteresis를 넣고, 실제 카메라에서 재획득 시간·overshoot·추적 유지율을 측정한다.

주의: 논문은 타일 경계 문제와 특정 데이터셋 의존성을 한계로 인정한다. 자사 현장에서는 overlap, temporal split, 다른 카메라·계절에 대한 외부 검증이 필요하다.
