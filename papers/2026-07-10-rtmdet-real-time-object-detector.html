RTMDet: 실시간 탐지기는 무엇을 균형 있게 설계해야 하는가

YOLO 계열의 속도-정확도 경쟁을 단순한 새 블록 하나가 아니라 backbone·neck 용량 배분, 큰 커널 depth-wise convolution, soft label assignment, 증강과 최적화의 공동 설계 문제로 풀어낸 실무형 논문이다.

한눈에 보는 요약

구조

5×5 depth-wise convolution으로 수용영역을 키우되 블록 수를 줄이고 폭을 늘려 병렬성을 확보한다.

학습

예측 box의 IoU를 soft label로 써서 분류 점수와 위치 품질을 함께 반영하는 동적 할당을 만든다.

운영

tiny부터 x까지 크기를 제공하고 TensorRT FP16, batch 1 기준으로 속도를 비교해 배포 관점이 명확하다.

수치 해석 주의: 논문의 지연시간은 NVIDIA RTX 3090, TensorRT 8.4.3, FP16, batch 1이며 객체 탐지 비교에서 NMS 시간은 제외됐다. 현장 RTSP 디코딩·전처리·NMS·네트워크 지연까지 포함한 FPS와 같지 않다.

왜 오늘 이 논문인가

금요일 주제인 실시간 구현·모델 최적화에 정확히 맞는다. 새 탐지는 원거리에서 몇 픽셀에 불과해 입력 해상도를 쉽게 낮출 수 없으므로, 단순히 모델을 축소하는 대신 어디에 계산량을 배치할지가 중요하다. RTMDet는 neck 용량, shared head, 동적 할당, augmentation cache를 하나씩 ablation해 이 판단 근거를 제공한다.

또한 공식 MMDetection 설정과 모델이 공개되어 YOLO baseline과 동일 데이터로 비교하기 쉽고, ONNX/TensorRT 배포 경로도 공식 문서에서 확인할 수 있다.

핵심 아이디어

표현 능력과 병렬성

큰 커널은 문맥을 넓게 보지만 layer가 늘면 지연이 증가한다. 블록 수를 줄이고 폭을 늘려 GPU 병렬 처리에 유리한 형태로 보정한다.

soft label assignment

분류 cost에 IoU soft target을 넣고 regression cost를 -log(IoU)로 계산해, 위치가 나쁜 고점수 예측을 좋은 positive로 고르는 오류를 줄인다.

기술 세부

DynamicSoftLabelAssigner

C = λ₁Ccls + λ₂Creg + λ₃Ccenter, 기본 가중치는 1:3:1이다. IoU soft label, log-IoU regression cost, soft center prior를 결합한다.

Cached Mosaic / MixUp

이전 이미지를 cache해 반복 I/O를 줄인다. 논문에서는 Mosaic 처리 시간이 약 3.6배, MixUp은 약 1.5배 빨라졌다고 보고한다.

2단계 300 epoch

앞 280 epoch는 cached Mosaic/MixUp, 마지막 20 epoch는 LSJ로 실제 분포에 가까운 영역에서 미세 조정한다.

배포 기준

FP16 TensorRT 8.4.3, cuDNN 8.2.0, RTX 3090, batch 1. RTMDet-tiny는 4.8M params와 COCO AP 41.1을 보고한다.

조류충돌방지 시스템 적용

PTZ 추적

RTMDet의 multi-scale box를 tracker에 넘기고, 화면 중심 대비 추적 box 중심 오차를 PTZ velocity command로 변환한다. 검출 confidence만 보지 말고 box 품질·track age를 함께 gate한다.

YOLO 데이터/학습

동일 train/val split에 YOLO와 RTMDet-s를 학습한다. tiny bird AP, recall, false alarm/hour, track fragmentation을 공통 지표로 둔다. Cached Mosaic 종료 시점을 조정해 실제 하늘 분포로 마무리한다.

서버 운영

RTSP decode → resize/normalize → TensorRT → NMS → tracker → PTZ 전체 p50/p95 지연을 기록한다. 모델 FPS가 아니라 end-to-end latency를 운영 SLO로 삼는다.

Hanwha Vision 연계 포인트

  • 카메라별 RTSP 프로파일 해상도와 bitrate를 고정하고 frame timestamp를 추적 결과에 보존한다.
  • PTZ 이동 중 motion blur 구간은 confidence threshold 또는 tracker update 정책을 별도로 둔다.
  • ONVIF/카메라 API의 pan·tilt 속도 제한, 명령 주기, backlash를 로그로 남겨 탐지 지연과 제어 지연을 분리한다.
  • RTMDet-R의 회전 box는 날개를 펼친 새의 방향 추정 실험에 활용할 수 있지만, PTZ 제어에는 안정적인 중심점이 우선이다.

즉시 실행할 실험 3개

  1. YOLO vs RTMDet-s 공정 비교: 640/960 입력, 동일 데이터·augmentation budget으로 tiny-bird AP와 전체 파이프라인 p95 지연을 비교한다.
  2. 고해상도 feature 실험: C3 head 유지/제거, 640 vs 1280, SAHI 사용 여부를 조합해 16px 이하 새 recall과 GPU 메모리를 측정한다.
  3. PTZ 폐루프 실험: 녹화 RTSP와 실제 카메라에서 detector+tracker+PTZ를 30분씩 운용해 중심 오차, 재포착 시간, false command/hour를 기록한다.

용어 정리

용어뜻
Depth-wise convolution채널별로 공간 convolution을 수행해 큰 커널의 계산량을 줄이는 방식.
Neckbackbone의 여러 해상도 특징을 합쳐 작은 객체와 큰 객체를 함께 다루는 모듈.
Dynamic label assignment고정 규칙 대신 현재 예측 품질에 따라 학습 positive를 동적으로 고르는 과정.
Soft label0/1 대신 IoU 같은 연속값으로 정답 품질을 표현한 target.
Flat-Cosine LR초반 학습률을 일정하게 유지한 뒤 후반에 cosine 형태로 낮추는 schedule.
End-to-end latency디코딩부터 PTZ 명령까지 사용자가 실제로 체감하는 전체 지연.

최종 체크리스트

  • □ 큰 커널만 추가하는 것이 아니라 depth·width·neck 비중을 함께 조정하는 이유를 설명할 수 있다.
  • □ soft label assignment가 분류 점수와 위치 품질을 결합하는 방식을 이해했다.
