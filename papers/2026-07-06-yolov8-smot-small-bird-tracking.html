YOLOv8-SMOT

작고 빠르게 움직이는 새를 UAV 시점 영상에서 추적하는 문제를, YOLOv8 기반 검출기와 외형 정보 없이 동작하는 추적기 조합으로 해결한 실무형 논문이다. 조류충돌방지 시스템에서 "먼 거리의 작은 새를 먼저 놓치지 않고, 잡은 뒤에는 추적 ID를 오래 유지하는 법"을 이해하는 데 직접 도움이 된다.

한눈에 요약

이 논문은 "새가 너무 작아서 Re-ID가 안 먹히는 상황"을 전제로 설계됐다. 그래서 외형 임베딩 대신 작은 객체를 더 잘 찾는 학습 전략과, 움직임 기반 연관 규칙을 강화하는 쪽에 집중한다.
1

고해상도 원본 이미지를 1280 x 1280 타일로 겹치게 잘라 학습하고, 추론은 다시 전체 프레임에서 수행한다.

2

타일 단위 강한 증강으로 배치 다양성을 높여, 먼 거리의 작은 새를 더 자주 보게 만든다.

3

추적기는 OC-SORT 계열을 쓰되, EMA 방향 유지와 확장 IoU + 거리 패널티로 작은 박스의 매칭 실패를 줄인다.

4

공개 테스트에서 YOLOv8-L 조합은 SO-HOTA 55.205, YOLOv8-S 조합은 17.61 FPS로 속도-정확도 균형을 보여준다.

5

조류충돌방지 현업에서는 "광각 검출 + 추적 안정화 + PTZ 제어 보수화"라는 구조로 바로 옮기기 좋다.

주의

확인된 공개 코드는 추론 중심 저장소다. 학습 재현은 논문 설명과 SMOT4SB 공식 baseline 저장소를 함께 봐야 한다.

왜 이 논문을 골랐나

실무 적합성 1: 일반 보행자 MOT가 아니라, 실제로 "작은 새"를 추적하는 문제를 다룬다. 조류충돌방지 회사에서 겪는 핵심 난제와 더 가깝다.

실무 적합성 2: 검출과 추적을 분리해서 개선한다. 그래서 기존 YOLO 파이프라인이 있는 팀이 일부만 차용해도 효과를 볼 가능성이 높다.

실무 적합성 3: 논문, 챌린지 페이지, baseline 저장소, 우승 코드 저장소까지 함께 검증할 수 있어서 후속 실험 설계가 쉽다.

바로 알아둘 한계: 이 논문은 UAV 기반 데이터에 맞춰져 있다. 풍력단지 고정 카메라나 PTZ 카메라는 시점과 배경이 다르므로, 절대 성능 숫자를 그대로 기대하면 안 된다.

추가 한계: 저자들도 극단적인 카메라 움직임과 긴 가림(occlusion)에서는 휴리스틱 기반 motion tracker의 한계를 인정한다.

우리 문제 논문이 주는 답 실무 해석
수 km 거리 새가 몇 픽셀만 보임 검출 학습을 타일 중심으로 바꿔 작은 객체 학습 빈도를 높임 원본 4K 라벨을 유지하면서도 학습 배치를 키울 수 있다
새가 서로 비슷해 Re-ID가 약함 외형 임베딩 없이 motion-only association을 강화 초기 버전에서는 복잡한 Re-ID보다 추적 규칙 안정화가 더 낫다
작은 박스는 IoU가 금방 0이 됨 박스 확장 + 중심 거리 패널티로 유사도 재설계 PTZ 추적 목표 선택 시 중심 이동량을 같이 보는 이유가 된다

핵심 아이디어

1. SliceTrain

고해상도 프레임을 겹치는 슬라이딩 윈도로 전부 덮어 자른 뒤, 각 타일에 Mosaic, 색상 변화, 기하 변환 같은 증강을 따로 건다. 핵심은 "랜덤 크롭"이 아니라 전체 픽셀을 빠짐없이 쓰는 결정적 분할이라는 점이다.

2. 학습은 잘라서, 추론은 원본으로

SAHI류 추론은 보통 잘랐다가 다시 합치느라 느려질 수 있다. 이 논문은 학습에서만 타일을 쓰고 추론은 전체 프레임에 바로 적용해서, 작은 새 감도와 실시간성을 같이 잡으려 한다.

3. 외형 없는 추적

새가 너무 작아 texture가 거의 없으니 Re-ID 특징이 불안정하다. 그래서 OC-SORT를 바탕으로 움직임 방향을 EMA로 매끈하게 유지하고, 박스 확장과 거리 패널티로 association을 강하게 만든다.

구성 요소 논문 설명 현업 번역
Deterministic Full-Coverage Tiling 경계 객체가 잘리지 않도록 겹치는 타일 생성 멀리 있는 작은 새를 놓치지 않는 학습 데이터 파이프라인
Slice-Level Stochastic Augmentation 타일마다 독립적으로 강한 증강 적용 안개, 역광, 흔들림, 구름, 로터 블러 유사 조건을 더 많이 학습
EMA Motion Direction 이전 속도 방향을 지수이동평균으로 안정화 PTZ 명령을 프레임마다 크게 흔들지 않도록 smoothing
Expanded IoU + Distance Penalty 작은 박스 특성에 맞춰 유사도 재정의 중심점 기준 추적과 gating 규칙 설계에 직접 차용 가능

기술 상세

항목 논문에서 확인한 내용 해석 포인트
문제 설정 UAV 시점의 작은 새 다중 추적(SMOT) 고정 카메라보다 어렵다. 이 설정에서 통하면 풍력 카메라에도 아이디어 차용 가치가 크다.
데이터셋 SMOT4SB, 총 211개 비디오 시퀀스(128 train / 38 val / 45 test) 원 논문과 공식 챌린지 페이지 숫자가 서로 맞아떨어진다.
사전 검출 학습 데이터 SOD4SB 기반 정적 이미지 47,260장 + 9,759장 subsets 영상 추적 전에 작은 새 검출기를 먼저 강하게 만드는 구조다.
타일 설정 1280 x 1280, overlap 20% 4K 카메라 학습을 유지하면서 작은 객체 픽셀 밀도를 높인다.
배치 효과 RTX 3090 한 장에서 full image batch 1 -> sliced batch 6 학습 안정성과 작은 객체 노출 빈도가 동시에 올라간다.
추적 파라미터 IoU threshold 0.25, decrement 0.08, track threshold 0.25, EMA α=0.8, expansion scale 2 현업 튜닝 시작점으로 쓸 만한 숫자다. 그대로 복사하지 말고 카메라별 재탐색 필요.
평가지표 SO-HOTA, SO-DetA, SO-AssA 작은 객체라 IoU보다 중심점 기반 Dot Distance가 더 낫다는 메시지가 핵심이다.
속도 YOLOv8-S 조합 17.61 FPS, YOLOv8-M 8.96 FPS, YOLOv8-L 5.70 FPS PTZ 추적은 S 또는 M부터 검토하는 편이 현실적이다.

조류충돌방지 시스템에 어떻게 적용할까

실무 한 줄 결론: 이 논문을 그대로 제품화하기보다, 광각 검출기 강화 + motion-only tracker 안정화 + 보수적인 PTZ 제어기 조합으로 분해해 가져오는 것이 가장 현실적이다.

조류충돌방지

풍력기 주변 위험 구역을 미리 정의한 뒤, 새가 해당 구역으로 진입하는 track만 별도 우선순위로 관리한다. 프레임 단발 검출보다 track 지속 시간 기준으로 경보를 거는 편이 거짓 경보를 줄인다.

PTZ 추적

PTZ를 매 프레임 직접 흔들기보다, 광각 카메라에서 track을 안정화한 뒤 0.5~1.0초 이상 유지된 목표에만 pan/tilt를 갱신한다. 작은 새는 박스가 흔들리므로 deadband와 max slew rate 제한이 중요하다.

YOLO 데이터/학습

현재 보유한 장거리 새 영상이 있다면 원본 4K 라벨은 유지하고, 학습 파이프라인에서만 겹치는 타일을 만든다. hard negative로 구름, 먼 드론, 곤충, 블레이드 끝 반사광을 반드시 포함한다.

실시간 서버 운영

영상 디코드, 검출/추적, PTZ 명령, 이벤트 저장을 한 프로세스에 섞지 말고 분리한다. track queue와 PTZ command queue를 나누면 지연과 폭주를 줄이기 쉽다.

Hanwha Vision 연계

한화비전 카메라에서는 RTSP 수신 경로와 PTZ 제어 경로를 분리해서 설계하는 편이 안전하다. 영상은 RTSP로 읽고, PTZ는 장비가 지원하는 ONVIF 또는 벤더 API를 모델별로 확인해 연결하는 방식이 운영상 깔끔하다.

압축/배포

논문 저자도 후속으로 QAT/PTQ를 언급한다. 현장 GPU가 약하면 먼저 YOLOv8-S나 M으로 재현하고, 그 뒤 ONNX/TensorRT/INT8 경량화를 붙이는 순서가 낫다.

배치 레벨 권장 구조 이 논문에서 가져올 요소
카메라 단 광각 감시 + 필요시 PTZ 추적 작은 새를 놓치지 않기 위한 고해상도 검출 학습
추론 단 검출과 추적 분리, track 안정도 계산 EMA 방향 유지, 확장 IoU, 거리 패널티
제어 단 위험 구역 진입, 지속 시간, confidence 기반 PTZ/경보 단발 detection보다 track 기반 의사결정

바로 해볼 실험 3가지

실험 1. SliceTrain A/B

목표: 현재 사내 YOLO 학습이 멀리 있는 새를 놓치는지 확인한다.

방법: 같은 데이터로 full image 학습 vs overlap tiling 학습을 비교한다.

성공 기준: 작은 객체 recall, 위험 구역 진입 track 수, false alarm/frame 개선.

실험 2. Motion-only Tracker

목표: appearance embedding 없이도 현장 영상에서 충분히 버티는지 확인한다.

방법: 현재 tracker와 OC-SORT + bbox expansion + distance penalty 조합을 동일 RTSP 녹화본에서 비교한다.

성공 기준: ID switch 감소, track fragmentation 감소, PTZ target handoff 안정화.

실험 3. PTZ Command Gating

목표: 검출 흔들림 때문에 PTZ가 흔들리는 현상을 줄인다.

방법: track age, center velocity, confidence EMA를 조합한 PTZ 발사 규칙을 오프라인 로그로 시뮬레이션한다.

성공 기준: 초당 PTZ 명령 수 감소, 목표 유지율 증가, 재획득 시간 단축.

용어 정리

용어 쉬운 설명
SMOT (Small Multi-Object Tracking) 작은 객체 여러 개를 프레임 간 이어서 추적하는 문제다. 작은 새 떼를 영상에서 계속 따라가는 상황을 생각하면 된다.
Tracking-by-Detection 먼저 매 프레임에서 검출하고, 그 결과를 이어 붙여 추적하는 방식이다.
SAHI / slicing 큰 이미지를 잘라 작은 객체를 더 잘 보이게 하는 접근이다. 이 논문은 추론보다 학습 쪽 slicing에 더 집중한다.
EMA (Exponential Moving Average) 최근 값을 더 크게 반영하는 평균이다. 새의 이동 방향을 너무 급격하게 바꾸지 않게 만드는 데 쓴다.
IoU (Intersection over Union) 예측 박스와 정답 박스가 얼마나 겹치는지 보는 지표다. 작은 새는 조금만 흔들려도 IoU가 급락한다.
SO-HOTA 작은 객체 추적용 평가지표다. 작은 박스에 취약한 일반 IoU 대신 점 거리 기반 유사도를 써서 더 공정하게 본다.

최종 체크리스트

  • 작은 새 검출 성능을 먼저 끌어올리고, 그 다음 tracker를 붙인다.
  • 학습 데이터는 원본 4K 라벨을 유지한 채 overlap tiling 버전을 추가한다.
  • PTZ는 detection 1프레임 기준이 아니라 stable track 기준으로만 제어한다.
  • RTSP 수신, 추적, PTZ 명령, 이벤트 저장 프로세스를 분리한다.
  • 현장 첫 재현은 YOLOv8-S 또는 M으로 시작하고, 이후 ONNX/TensorRT/INT8을 검토한다.
  • SMOT4SB 숫자는 참고용이고, 실제 풍력/조류충돌 현장 데이터로 반드시 재검증한다.
