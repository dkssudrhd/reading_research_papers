한눈에 보는 요약

Wake Vision은 초저전력 비전(TinyML)에서 “모델을 더 작게 만드는 것”만큼 학습·평가 데이터의 품질과 현장 조건별 측정이 중요하다는 점을 보이는 연구다. 사람 이진 분류 데이터셋이지만, 멀리 있는 새처럼 화면 점유율이 작은 대상에 대해 입력 해상도, 라벨 정제, INT8 양자화, 조도별 성능을 어떻게 검증할지에 직접적인 방법을 준다.

왜 이 논문을 골랐나

화요일의 작은 객체 탐지 주제에 맞춘 선택이다. 풍력단지 조류 영상은 “전체 평균 AP”가 좋아도 먼 거리·역광·안개 프레임에서 무너질 수 있다. Wake Vision은 거리·조도·묘사물 등 세분 subset을 만들고, 모델 용량·입력 크기·양자화가 각 subset에 미치는 영향을 분리해 본다. RTSP 서버와 엣지 장비의 배포 결정을 데이터로 할 수 있게 해 준다.

거리·조도·묘사물 같은 배치 조건별 benchmark가 있어야 작은 대상의 실패 원인을 찾을 수 있다.

기술 세부사항

논문 관찰근거/수치조류 시스템 해석
Quality가 Large보다 우세MobileNetV2-0.25에서 bounding-box 기반 Quality가 image-level Large보다 높은 정확도새 box 검수·hard negative의 품질을 먼저 올린다.
먼 대상은 해상도 병목거리 subset에서 입력 해상도 증가 효과가 모델 크기 증가보다 컸다.4K 원본→tile/고해상도 crop의 ROI를 수치로 검증한다.
INT8의 이점원문 case study에서 모든 subset의 성능 저하는 ±0.004 F1 이내였다.현장 bird 데이터에서도 FP16/INT8을 subset별로 재검증한다.
그레이스케일 주의밝은 장면에서 RGB→grayscale의 영향이 더 컸다.역광·눈부심 조건의 RGB 제거는 별도 gate가 필요하다.

조류충돌방지 시스템 적용

bird detection / YOLO 데이터

라벨을 영상 단위로 분리하고, 실제 오탐(구름, 전선, 잎, 곤충)을 hard-negative pool로 유지한다. ‘Large’식 자동수집 데이터와 ‘Quality’식 검수 데이터를 분리해 ablation한다.

PTZ 추적

거리 bucket(예: box 면적 비율)별 recall과 추적 지속시간을 기록한다. 멀리 있는 새에서 recall이 낮으면 PTZ 제어 이득보다 고해상도 crop/재탐지 이득이 먼저일 수 있다.

RTSP·서버·엣지

RTSP decode→detect→track→PTZ API 지연을 따로 측정한다. ONNX/TensorRT INT8은 전체 평균뿐 아니라 야간·역광·먼 대상 subset의 recall을 통과 기준으로 삼는다.

바로 실행할 실험 3가지

  1. Bird Quality/ Large 분리: 자동 pseudo-label 풀과 사람이 재검수한 2k–5k 프레임을 같은 YOLO recipe로 학습해 APsmall, 먼 거리 recall, 오탐률을 비교한다.
  2. 거리×해상도 matrix: 전체 1280 입력, 4K 2×2 tile, tracking crop을 box-area bucket별로 비교하고 FPS·GPU·PTZ 재획득률을 함께 기록한다.
  3. 배포 회귀시험: PyTorch FP32, ONNX FP16, TensorRT INT8에 대해 조도·날씨·거리별 고정 validation을 만들고, 어느 subset도 recall이 기준보다 낮아지면 배포를 막는다.

용어

TinyML
센서 가까이의 MCU·저전력 가속기에서 작은 모델을 실행하는 머신러닝.
Fine-grained benchmark
전체 평균 대신 거리·조도처럼 운영 조건별로 나눈 평가 집합.
Knowledge distillation
큰 teacher의 soft label을 작은 student 학습에 쓰는 지식 증류.
INT8 quantization
가중치/연산을 8비트 정수로 바꿔 모델 크기와 지연을 줄이는 최적화.
Depiction
실제 대상이 아닌 그림·렌더링·화면 속 인물처럼 혼동 가능한 묘사물.
