YOLOv10: NMS 없는 실시간 조류 탐지 파이프라인

YOLOv10은 학습 때는 one-to-many의 풍부한 감독을 쓰고, 추론 때는 one-to-one 예측만 남겨 NMS를 제거한다. 현장 RTSP→탐지→추적→PTZ 명령 경로에서 변동 지연을 줄이는 데 특히 유용하다.

한눈에 요약

후처리 제거
추론용 one-to-one head는 객체당 하나의 상자만 내므로 NMS가 필요 없다.
효율 설계
가벼운 분류 head, 공간·채널 분리 downsampling, rank-guided CIB로 중복 연산을 줄인다.
정확도 설계
깊은 단계의 large-kernel convolution과 저해상도 PSA로 문맥을 보강한다.

왜 이 논문인가

조류충돌방지 시스템은 탐지기 자체 FPS보다 프레임 도착 시각의 흔들림과 PTZ 명령까지의 총 지연이 중요하다. NMS는 장면별 후보 수에 따라 후처리 시간이 달라질 수 있다. YOLOv10은 이를 제거한 뒤에도 YOLO 계열의 CNN 장점을 유지하며, 공식 구현·ONNX 계열 생태계·추적기 연동 사례가 있어 검증 가능한 비교 기준이 된다.

핵심 아이디어

일관된 이중 할당(consistent dual assignments): 학습 중 one-to-many(o2m) branch는 여러 positive로 backbone/neck을 충분히 학습시킨다. 별도 one-to-one(o2o) branch는 같은 구조·목적함수를 공유하되 객체마다 한 prediction을 선택한다. 배포 때 o2m branch를 버리므로 추가 추론비용이 없다. 두 branch의 matching metric을 같게 해 선택 순위와 분류 감독의 간극을 줄이는 것이 핵심이다.

기술 세부

Matching metric
m = s^α · IoU(b, b̂)^β · 1{anchor∈box}. 분류 점수와 위치 품질을 함께 써 o2m/o2o top 선택을 일치시킨다.
Downsampling
먼저 1×1 pointwise convolution으로 채널을 바꾸고, 다음 depth-wise stride 2로 공간을 줄인다. 정보를 보존하며 비용을 낮추려는 순서다.
CIB
저렴한 depth-wise로 spatial mixing, pointwise로 channel mixing을 하는 compact inverted block. intrinsic rank가 낮은 stage부터 대체한다.
PSA
채널 절반에만 MHSA/FFN을 적용하고, 가장 낮은 해상도 stage 뒤에 둔다. 전역 문맥은 얻되 attention의 제곱 비용을 제한한다.

조류충돌방지 시스템 적용

PTZ 추적
검출 box 중심과 화면 중심 차이를 PID 또는 속도 명령으로 바꾸되, 탐지 1프레임만으로 움직이지 말고 track age·IoU·confidence를 gate에 넣는다. NMS-free 출력은 제어 지연 분산을 줄일 후보가 된다.
YOLO 데이터·학습
작은 새, 먼 거리, 역광, 새 아닌 점(먼지·곤충·구름)을 별도 slice로 유지한다. o2o가 객체당 하나를 내더라도 작은 객체 recall, false alarm/hour, ID switch를 분리 측정한다.
실시간 서버
RTSP decode→timestamp→resize→ONNX/TensorRT→tracker→ONVIF PTZ 명령의 p50/p95를 기록한다. GPU 추론 ms와 NMS 제거 효과를 전체 경로에서 분리해 본다.

Hanwha Vision 연계: 프로파일별 RTSP 해상도·GOP·bitrate와 카메라 내부 timestamp를 로그로 남긴다. 이동 중 blur에는 검출 threshold를 올리기보다 tracker 예측만 허용하는 구간을 실험하고, PTZ 속도/가감속/명령 주기를 별도 측정한다.

즉시 실행할 실험 3개

  1. NMS 영향 분해: 동일 bird validation에서 YOLOv8 + NMS와 YOLOv10의 detector latency, 후보 수, end-to-end p95, false command/hour를 비교한다.
  2. 작은 새 slice 평가: 16px 이하/16–32px/역광/비조류 hard negative로 나눠 recall·AP·재포착 시간을 기록한다. 640과 960/타일링 조건도 고정한다.
  3. 폐루프 PTZ: 녹화 RTSP 후 실제 Hanwha 카메라에서 30분씩 구동해 중심 오차, target lost 후 재포착 시간, 명령 초과 빈도를 YOLOv8와 비교한다.

수치 주의: 논문 지연은 공식 사전학습 모델, T4 GPU, TensorRT FP16 기준이다. 현장의 디코딩·네트워크·추적·PTZ 시간을 포함하지 않는다.

용어 정리

용어뜻
NMS (Non-Maximum Suppression)겹치는 후보 상자 중 하나를 남기는 후처리.
one-to-many / one-to-one정답 하나에 여러 positive를 배정하는 방식 / 하나만 배정하는 방식.
Intrinsic rank특징 또는 가중치의 실질적 독립 정보량을 수치적으로 본 값. 낮으면 중복 가능성을 시사한다.
APprecision-recall 곡선 아래 면적의 평균. 작은 새 recall이나 오경보율을 대체하지는 않는다.
