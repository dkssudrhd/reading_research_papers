요약

이 논문은 고고도 UAV에서 지상 차량을 놓치지 않도록 PTZ의 pitch·yaw·roll·zoom을 강화학습(PPO)으로 선택한다. 핵심은 이미지 전체를 정책에 반복 입력해 품질을 추정하지 않고, 관측 상태에서 물체 인식 확률을 예측하는 상태 인식 모델을 먼저 학습해 보상으로 쓰는 점이다. 현장 조류 시스템에서는 UAV 대신 고정형 Hanwha PTZ, 차량 대신 새로 바꿔 “화면 중앙 유지 + 검출 신뢰도 유지”라는 제어 목표를 검증할 수 있다.

왜 선택했나

목요일 주제 적합
PTZ의 pan/tilt/zoom 제어를 논문의 중심 문제로 다룬다.
현장 지연을 모델링
대상 소실 후 재탐색과 제어 안정성을 명시적으로 평가한다.
적용 한계
원 실험은 고고도 UAV-차량 시뮬레이션이다. 새·고정형 PTZ·실제 RTSP 지연으로 재검증이 필요하다.

핵심 아이디어

  1. 상태 인식 모델: 거리, 방위각, pitch, 배율, 대상 존재 플래그에서 YOLOv4가 낸 검출 확률을 근사한다.
  2. 보상 형성(reward shaping): 검출 확률이 높은 관측 상태와 화면 중심에 가까운 대상에 높은 보상을 준다.
  3. 세계 모델: UE 시뮬레이터의 내부 함수를 근사하고 보정 모델을 더해 RL rollout을 빠르게 만든다.
  4. 개선 PPO: 장기 소실 때 누적 보상이 임계값 아래면 에피소드를 끝내어 재탐색 학습을 유도한다.

기술 세부

관측 상태는 sₜ=[xₜ/w, yₜ/h, zₜ/zmax, pxₜ/(w·h), fₜ]ᵀ 이다. 여기서 x,y는 물체와 화면 중심의 상대 위치, z는 배율, px는 물체 픽셀 면적, f는 대상 존재 여부다. 행동은 pitch·yaw·roll·zoom 각각 −2부터 2의 이산값이다. 전체 보상은 rₜ=m rᵏₜ+n rᵛₜ로, rᵏ는 상태 인식 모델의 확률, rᵛ는 중심까지의 정규화 유클리드 거리(대상 소실이면 −1)다. 논문은 UE에서 모은 24,000장과 사전학습 YOLOv4를 사용해 상태 인식 모델을 학습했고, 세계 모델은 UE 대비 약 10배 학습 효율 향상을 보고한다.

조류 충돌 방지 적용

Bird detection
YOLO의 bird confidence·bbox 크기·위치를 상태에 넣고, confidence 하락을 줌/추적 품질 저하 신호로 쓴다.
PTZ tracking
ONVIF 또는 Hanwha Vision API의 상대 이동 명령은 행동 공간으로, 카메라 응답 시간은 세계 모델의 보정 대상으로 둔다.
RTSP / 서버
RTSP timestamp, decode, detector, command, 움직임 완료를 모두 기록해 폐루프 지연을 관측한다.
YOLO / edge
학습에는 YOLO confidence를 교사 신호로 쓰되, 배포 시에는 ONNX/TensorRT detector와 제어 정책의 지연 예산을 합산한다.

바로 할 실험 3개

  1. 규칙 기반 기준선: bbox 중심 오차에 비례해 pan/tilt를, bbox 면적 오차에 비례해 zoom을 움직이는 PID를 먼저 만든다.
  2. 지연 보정 실험: Hanwha PTZ 명령부터 RTSP 프레임 반영까지의 분포를 수집하고, 100/300/500 ms 지연에서 PID·PPO 재획득률을 비교한다.
  3. 보상 A/B: 중심 거리만 쓴 보상과 0.6×bird confidence − 0.4×중심거리 보상을 비교해 검출 가능한 픽셀 크기 유지 효과를 측정한다.

측정: target-in-FoV 비율, 평균 중심 오차, 재획득 시간, bird mAP/recall, 명령 빈도, end-to-end 지연 p95.
