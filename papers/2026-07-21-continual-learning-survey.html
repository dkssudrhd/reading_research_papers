한눈에 보는 요약

지속 학습(Continual Learning, CL)은 새 데이터를 순차적으로 배우면서 이전 지식을 잊지 않게 하는 방법론이다. 이 서베이는 핵심 목표를 안정성–가소성(stability–plasticity) 균형, 과제 내/과제 간 일반화, 자원 효율로 정리하고, regularization·replay·optimization·representation·architecture 기반의 다섯 계열을 체계화한다.

왜 이 논문을 골랐나

풍력단지·공항·계절·카메라 설치 위치가 바뀌면 조류 탐지 데이터 분포도 바뀐다. 새 현장 데이터만으로 YOLO를 계속 fine-tuning하면 과거 현장의 성능을 잃는 catastrophic forgetting이 생길 수 있다. 이 논문은 기존 모델을 재학습하지 않고도 갱신할 때 필요한 실험 설계의 지도 역할을 한다.

핵심 아이디어

안정성 vs 가소성

기존 현장을 기억하면 새 현장 적응이 늦고, 새 현장에 과적응하면 기존 현장을 잊는다.

다섯 전략

규제, 과거 샘플 재생, gradient 제약, 표현 학습, 구조 확장으로 망각을 줄인다.

평가의 핵심

마지막 모델만 보지 말고 각 과제 정확도 행렬로 평균 정확도·망각·forward transfer를 함께 본다.

기술 세부사항

계열핵심조류 시스템 해석
Regularization이전 모델의 중요한 parameter/출력을 크게 바꾸지 않음기존 현장 detector의 feature·logit distillation을 추가
Replay과거 대표 샘플 또는 생성 샘플을 새 데이터와 재학습카메라·계절별 hard case를 작고 균형 잡힌 buffer로 유지
Optimization과거 과제 성능을 해치지 않는 gradient 방향을 선택연산 여유가 있을 때 새 fine-tune의 gradient interference 측정
Representation분포 변화에 덜 민감한 표현을 학습자기지도 pretraining과 domain augmentation을 비교
Architectureadapter·mask 등 과제별 parameter를 분리현장별 adapter는 관리가 쉽지만 모델·운영 복잡도가 증가

조류충돌방지 시스템 적용

bird detection / YOLO

새 현장 fine-tune 때 이전 현장 프레임을 10–30% replay하고, 과거 checkpoint의 distillation loss를 비교한다.

PTZ tracking

카메라 firmware·zoom preset 변경도 domain shift로 기록한다. box AP뿐 아니라 track 유지율과 재획득 시간을 과제별 matrix로 남긴다.

RTSP / edge

buffer에는 원본 비디오가 아니라 개인정보·보존 정책을 통과한 대표 crop/feature를 사용한다. ONNX/TensorRT 엔진도 현장별 회귀 검증 뒤 교체한다.

바로 실행할 실험 3가지

  1. 계절 순차 학습: 봄→여름→가을 데이터를 순서대로 학습하고 각 계절 validation APsmall 행렬을 만든다.
  2. replay 비율 ablation: 새 데이터만, 10%, 30% 과거 replay를 같은 epoch budget에서 비교해 평균 정확도와 망각량을 기록한다.
  3. PTZ domain split: 고정형/줌 확대/추적 중 preset별로 task를 나누고 detector와 tracker의 forgetting을 분리 측정한다.

용어

Catastrophic forgetting
새 과제 학습 뒤 이전 과제 성능이 급격히 하락하는 현상.
Replay
과거 실제 샘플·특징·생성 샘플을 다시 학습에 섞는 방법.
Class-Incremental Learning (CIL)
학습 때만 과제 식별자가 있고, 추론 때는 모든 클래스 중 예측해야 하는 설정.
Domain-Incremental Learning (DIL)
라벨 공간은 같고 입력 분포만 바뀌는 설정. 카메라·계절 변화와 가깝다.
Forward/Backward transfer
이전 학습이 새 과제를 돕는 효과/새 학습이 과거 과제를 개선·훼손하는 효과.
