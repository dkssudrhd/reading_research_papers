SMOT4SB Challenge Report

작은 새를 UAV 영상에서 여러 프레임에 걸쳐 추적하려면, 검출기 하나만 잘 돌리는 것으로는 부족하다. 이 논문은 조류 추적용 데이터셋, 작은 객체용 평가 지표, 상위권 해법들을 한 번에 제공해서, 조류충돌방지 시스템의 검출-추적-평가 루프를 다시 설계할 때 직접 참고하기 좋다.

한눈에 요약

이 논문은 "작은 새 추적"을 연구 주제로만 다루지 않고, 실제로 재현 가능한 연구 인프라로 정리한다. 즉, SMOT4SB 데이터셋으로 문제를 정의하고, SO-HOTA라는 작은 객체 친화 metric으로 평가하며, YOLOX + OC-SORT baseline과 상위 5개 팀의 설계를 함께 보여 준다.
핵심 1

새가 몇 픽셀 수준으로 작을 때는 IoU가 지나치게 예민해져 평가가 흔들린다. 그래서 이 논문은 중심점 거리 기반의 DotD를 써서 SO-HOTA를 정의한다.

핵심 2

데이터셋은 211개 비디오, 108,192 프레임, 371,690 bird instance, 2,240 ID로 구성된다. 환경도 숲, 강, 들판, 도시, 공원까지 넓다.

핵심 3

우승팀은 YOLOv8-SOD + YOLOv8-SMOT 조합으로 private test SO-HOTA 50.59를 기록했고, baseline 9.90 대비 5.1배 수준의 개선을 보였다.

핵심 4

검출은 SliceTrain처럼 작은 객체를 더 자주 보게 만드는 학습 전략이 중요했고, 추적은 appearance보다 motion 안정화와 adaptive similarity가 더 중요했다.

핵심 5

조류충돌방지 시스템에서는 "광각 감시 카메라에서 작은 새를 놓치지 않는 검출"과 "PTZ/추적 단계로 넘길 때 흔들리지 않는 ID 유지"라는 두 과제를 동시에 설계하는 기준으로 읽어야 한다.

주의

원문 서론에는 데이터셋 프레임 수가 183,192로 적힌 부분이 있지만, abstract, Table 1, Table 2, challenge page는 모두 108,192를 가리킨다. 본 노트는 상호 일치하는 수치인 108,192를 채택했다.

왜 이 논문을 골랐나

수요일 주제 적합: 오늘 테마가 object tracking / MOT인데, 이 논문은 그냥 MOT가 아니라 조류용 small multi-object tracking을 정면으로 다룬다.

실무 적합: bird strike avoidance, ecological monitoring, UAV perception을 명시적으로 언급한다. 회사 업무와 논문 주제의 간격이 짧다.

재현성: 공식 baseline 코드, dataset 링크, challenge page, leaderboard가 모두 확인된다. 메서드만 추상적으로 설명하는 논문보다 훨씬 실용적이다.

번역 가능성: arXiv 라이선스가 CC BY-SA 4.0로 확인돼, 오늘은 공개 저장소용 전문 번역까지 깔끔하게 진행할 수 있다.

바로 이식하면 안 되는 부분: 데이터는 UAV 영상이라 고정형 풍력단지 카메라나 Hanwha Vision PTZ와는 카메라 운동 모델이 다르다.

그래도 중요한 이유: PTZ가 빠르게 pan/tilt할 때도 장면 전체가 움직이므로, 논문이 말하는 motion entanglement는 PTZ 추적의 급회전 구간과 개념적으로 연결된다.

메타데이터 주의: challenge page의 초기 설명에는 일부 split 수치가 TBA로 남아 있다. 최종 숫자는 challenge report 본문/Table 2 기준으로 읽는 것이 안전하다.

핵심 아이디어

1. 연구 인프라를 한 번에 준다

대부분의 논문은 "새로운 모델" 하나만 제안한다. 이 논문은 다르다. 데이터셋, 평가 지표, baseline, 상위권 해법을 함께 제공해 후속 연구 출발점을 만든다.

2. 작은 객체에는 IoU만으로 부족하다

새가 10~20픽셀 수준이면 박스가 1~2픽셀만 흔들려도 IoU가 급락한다. 그래서 중심점 거리 기반의 DotD를 쓰고, 이를 HOTA에 결합한 SO-HOTA를 제안한다.

3. 검출과 추적을 따로 강화한다

우승 해법은 검출기에서는 SliceTrain으로 작은 객체 학습을 강화하고, 추적기에서는 EMA와 adaptive similarity로 motion-only association을 더 안정화했다.

4. motion entanglement를 문제 중심에 둔다

카메라와 새가 동시에 움직이는 상황을 별도 challenge attribute로 강조한다. 이 관점은 PTZ 추적에서도 매우 중요하다.

5. 성능표만이 아니라 설계 패턴을 준다

zhwa2003의 false positive filtering, elsalabA의 SAHI + Copy-Paste, sgm의 adaptive WBF, xmba15의 motion compensation처럼 서로 다른 전략을 비교할 수 있다.

6. 실무에서는 평가 설계를 바꾸라는 논문

이 논문의 가장 큰 메시지는 "모델을 바꾸기 전에 metric부터 점검하라"이다. 작은 새 추적은 잘못된 metric을 쓰면 개선 여부 자체를 오판할 수 있다.

SO-HOTA 핵심 식

DotD(A, B) = exp(-d(A, B) / s)
d(A, B) = sqrt((x_A - x_B)^2 + (y_A - y_B)^2)
s = sqrt((Σ_i Σ_j w_ij * h_ij) / (Σ_i N_i))

DetA_α = |TP| / (|TP| + |FN| + |FP|)
AssA_α = (1 / |TP|) Σ_{c in TP} |TPA(c)| / (|TPA(c)| + |FNA(c)| + |FPA(c)|)
SO-HOTA_α = sqrt(DetA_α * AssA_α)
SO-HOTA = (1 / 19) Σ_{α in {0.05,...,0.95}} SO-HOTA_α

원문 challenge page의 수식 표기를 그대로 옮겼다. 핵심은 IoU 대신 중심점 거리 기반 similarity를 쓴다는 점이다.

공식 baseline

논문 baseline은 YOLOX detector + OC-SORT tracker의 2단 구조다. 따라서 상위권 방법을 읽을 때도 "검출기 개선"과 "association 개선"을 분리해서 봐야 한다.

우승 해법은 여기에 YOLOv8-SOD, SliceTrain, EMA motion, adaptive similarity를 얹었다.

다른 팀들도 공통적으로 false positive 억제, small-object friendly training, motion compensation, ensemble fusion 중 적어도 하나를 강하게 썼다.

SliceTrain으로 작은 새 검출을 강화하고, OC-SORT 위에 EMA와 adaptive similarity를 넣었다. 가장 적은 파라미터로 최고 점수를 냈다.

전봇대 같은 urban false positive가 많은 장면을 겨냥해 intersection-based ensemble로 오탐을 줄였다. "특정 배경 구조물에 대한 음성 prior"라는 발상이 유용하다.

각각 Copy-Paste + SAHI, adaptive WBF multi-YOLO ensemble, motion-compensated OC-SORT를 썼다. 결국 핵심은 작은 객체 검출 안정화와 camera motion 처리다.

조류충돌방지 시스템 적용 메모

bird detection

새가 매우 작을 때는 detection miss가 tracking 실패보다 먼저 온다. 따라서 먼저 small-object friendly detector를 만들고, 추적기는 그 위에 얹는 구조가 맞다.

PTZ camera tracking

이 논문은 UAV를 다루지만, PTZ 급회전 구간도 장면 전체가 흔들리므로 motion entanglement와 닮아 있다. PTZ에서도 ego-motion compensation이 중요하다.

YOLO data / training

bird-only 데이터가 적더라도 SliceTrain, Copy-Paste, patch-level 학습으로 작은 새 출현 빈도를 의도적으로 늘릴 수 있다.

RTSP / 실시간 서버

RTSP 서버에서는 full-frame detector + lightweight tracker 조합이 현실적이다. 우승팀이 YOLOv8-S에서 17.61 FPS를 보였다는 점은 edge 서버 기준선이 된다.

ONNX / TensorRT

논문은 quantization-aware training과 post-training quantization 가능성을 직접 언급한다. 즉, 정확도 논문이면서 배포 단계까지 생각하고 있다.

Hanwha Vision / RTSP / PTZ 연동

Hanwha Vision 카메라라면 광각 채널에서 bird candidate를 얻고, SO-HOTA 스타일 추적 안정성 기준을 만족할 때만 PTZ preset 이동 또는 continuous pan/tilt를 걸도록 설계할 수 있다.

현재 파이프라인 문제 논문에서 가져올 요소 실무형 해석
원거리 새가 너무 작아 첫 박스가 불안정함 SliceTrain / SAHI 계열 학습 및 추론 광각 입력은 patch-level 학습 또는 ROI slicing으로 개선
카메라 pan/tilt 순간 추적 ID가 쉽게 깨짐 motion compensation, center-distance 중심 association PTZ 제어 구간에서 appearance보다 motion prior 강화
IoU가 작게 나와도 체감상 잘 따라가는데 평가가 나쁨 SO-HOTA / DotD 사내 평가 지표에 center-distance 계열 보조 score 추가

지금 바로 해 볼 실험 3개

실험 1. SO-HOTA 흉내 지표 만들기

목표: 현재 YOLO + tracker 결과를 IoU 기반으로만 평가할 때 생기는 왜곡을 확인한다.

방법: 사내 bird tracking 로그에서 GT 중심점과 prediction 중심점의 거리 기반 score를 추가 계산한다.

관찰 포인트: IoU는 나쁜데 체감상 괜찮은 시퀀스가 얼마나 많은지 본다.

실험 2. bird patch 재학습

목표: RTSP 광각 새 탐지의 first-hit miss를 줄인다.

방법: 기존 bird dataset을 1280 또는 960 패치로 겹치게 잘라 재학습하고, full-frame inference 성능을 비교한다.

관찰 포인트: recall, false positive, TensorRT latency를 함께 본다.

실험 3. PTZ motion compensation

목표: PTZ 급이동 구간의 ID switch를 줄인다.

방법: 연속 프레임에서 keypoint matching으로 affine 보정을 한 뒤, 보정 전후의 association stability를 비교한다.

관찰 포인트: ID switch, track fragmentation, PTZ re-center 시간.

용어 정리

영문 용어 한국어 설명
Small Multi-Object Tracking (SMOT)작은 객체가 여러 개 동시에 움직일 때, 각 객체의 위치와 ID를 프레임 간에 유지하는 문제
Motion Entanglement카메라 움직임과 대상 움직임이 서로 섞여 보여서, 실제 대상 운동을 분리하기 어려운 상태
Dot Distance (DotD)박스 겹침보다 중심점 간 거리를 더 직접적으로 보는 similarity score
SO-HOTA작은 객체 추적을 위해 HOTA를 DotD 기반으로 바꾼 평가 지표
SO-DetA / SO-AssA검출 정확도와 연관 정확도를 각각 측정하는 SO-HOTA 하위 점수
SliceTrain고해상도 이미지를 작은 패치로 나눠 학습해 tiny object 학습 효율을 높이는 전략
Adaptive WBF고정 가중치 대신 프레임별 confidence를 바탕으로 box fusion 가중치를 조정하는 방식

  • YOLO 학습 개선 포인트를 정했는가? patch-level 학습, SliceTrain, Copy-Paste를 우선 시험한다.
  • PTZ 추적 보강 포인트를 정했는가? ego-motion 보정과 association 안정화 실험을 분리한다.
  • 실시간 배포 기준을 정했는가? FPS, parameter, quantization 가능성을 함께 본다.
